# TestCaseEditor App - Workspace Layout Reference

## 5-Workspace Pattern Layout (Top to Bottom)

1. **Title Workspace** (top)
   - Example: "Test Case Generator"
   - Shows current domain/mode title

2. **Notification Workspace** (under title)
   - Status information bar
   - Example: "LLM: Disconnected", "Method: No requirement selected", "Requirements w/Test Cases: 0/0"

3. **Header Workspace** (below notification) 
   - Context-specific header content
   - May appear as empty/blank orange bar when no header content

4. **Main Workspace** (largest central area)
   - Primary content area
   - Example: Welcome content, instructions, main UI

5. **Navigation Workspace** (bottom)
   - Domain-specific navigation controls
   - Example: "1 / 0", "No requirements loaded", Search box

## Side Menu Workspace (Left)
   - Global navigation menu
   - Example: Project, Requirements, LLM Learning, Test Case Creation
